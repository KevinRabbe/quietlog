# QuietLog — Update Philosophy

## Goal

Updates should feel respectful to users.

QuietLog should avoid the modern pattern of:
- giant updates
- vague patch notes
- unnecessary downloads
- constant feature churn

## Patch Notes

Patch notes should clearly explain:
- what changed
- what was fixed
- what improved

Avoid vague messages like:
- general improvements
- bug fixes and optimizations

## Update Size Philosophy

Prefer:
- focused updates
- small improvements
- efficient asset usage
- minimal unnecessary dependencies

## Long-Term Philosophy

QuietLog should grow through:
- stable iteration
- meaningful improvements
- reliability improvements

not through endless feature bloat.
