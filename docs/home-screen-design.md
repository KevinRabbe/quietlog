# QuietLog — Home Screen Design

## Goal

The Home screen should be a calm daily dashboard.

Core idea:

> Home = What matters next.

The Home screen should not become a feed, statistics page, or cluttered productivity dashboard.

---

## Home Screen Sections

## 1. Today

Shows things relevant today.

Can include:

- reminders due today
- app events happening today
- active shopping lists

Example:

```text
Today
- Change tires — 15:00
- WoW raid — 20:00
- Weekly groceries — 4/12 done
```

---

## 2. Next Up

Shows the next few upcoming items.

Recommended limit:

- 3 to 5 items

Purpose:
Give the user quick orientation without overwhelming them.

Example:

```text
Next Up
- Change tires — Today 15:00
- WoW raid — Friday 20:00
- Monthly household — Sunday
```

---

## 3. Active Shopping

Shows active shopping list progress.

Example:

```text
Shopping
Weekly groceries — 4/12 done
```

The user should be able to tap the card and go to Shopping.

---

## 4. Next App Event

Shows the next app-linked event/reminder.

Example:

```text
Next App Event
World of Warcraft — Raid — Friday 20:00
Reminder: 20 min before
```

The user should be able to tap the card and go to Apps.

---

## 5. Quick Actions

Simple quick actions:

- Add reminder
- Add shopping item/list
- Add app event

These actions should reduce friction but not dominate the screen.

---

## 6. Quiet Status

Small optional status card.

Purpose:
Reinforce trust and app identity.

Example:

```text
QuietLog is offline.
No account. No cloud. No ads.
```

This should stay small and calm.

---

## Recommended V1 Layout

```text
QuietLog

Today
[Change tires        15:00]
[WoW raid            20:00]

Next Up
[Monthly household   Sunday]

Shopping
[Weekly groceries    4/12 done]

Quick Actions
[Reminder] [Shopping] [App Event]
```

---

## Empty State

If there is nothing scheduled:

```text
Nothing urgent right now.
Add a reminder, shopping list, or app event when needed.
```

The empty state should feel calm, not like the app is begging for engagement.

---

## Do Not Add

Do not add:

- social feed
- streaks
- gamification
- AI suggestions
- ads
- analytics dashboard
- complicated statistics
- calendar month view in V1
- overloaded widget grid

---

## Design Rules

- show only what matters next
- prefer short lists
- keep sections tappable
- avoid notification-like pressure
- avoid visual clutter
- keep dark theme readable
- make Home useful even with little data

---

## Main Rule

The Home screen should help the user feel oriented, not busy.
