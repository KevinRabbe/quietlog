# Contributing To QuietLog

## Goal

Keep QuietLog:

- reliable
- lightweight
- maintainable
- offline-first

## Main Rules

- Keep commits focused.
- Avoid feature creep.
- Do not add unnecessary dependencies.
- Prefer readable code over clever code.
- Keep the app buildable.

## Forbidden Additions

Do not add:

- cloud sync
- accounts
- AI features
- ads
- analytics/tracking
- subscriptions
- social systems
- multiplayer systems

## Before Opening PRs

Ensure:

- app builds successfully
- no obvious crashes
- changes are scoped
- commits are understandable

## Important

Reminder reliability is one of the highest priorities of QuietLog.
