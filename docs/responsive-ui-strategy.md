# QuietLog — Responsive UI Strategy

## Goal

QuietLog should feel consistent across:

- small Android phones
- large phones
- Pixel devices
- future tablets

The UI should remain:

- readable
- touch-friendly
- stable
- predictable

---

## Core Strategy

Use:

- Jetpack Compose
- responsive layouts
- flexible spacing
- adaptive containers

Avoid:

- fixed pixel sizes
- overcrowded screens
- desktop-style complexity

---

## Layout Philosophy

Prefer:

- vertical scrolling
- cards
- lists
- bottom navigation

Avoid:

- tiny buttons
- deep nested menus
- overly complex multi-column layouts

---

## Touch Targets

Minimum touch target:
48dp

Important for:

- shopping lists
- reminder actions
- one-handed use

---

## Main Rule

Consistency is more important than maximizing every screen size.
