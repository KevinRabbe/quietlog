# QuietLog — Marketing And Release Plan

## Goal

Build trust before launch without hype, spam, or overpromising.

QuietLog should be marketed as:

> A private offline planner for people who hate notification spam.

Core promise:

- no ads
- no cloud
- no account
- no AI
- no subscription
- only user-created reminders

---

# Release Strategy

## Phase 1 — Development Visibility

Goal:
Start showing the project early while making it clear that reliability comes before launch speed.

Content:
- app philosophy
- early screenshots
- reminder reliability progress
- offline-first architecture
- no ads / no cloud / no account positioning

Do not:
- promise release dates too early
- show fake mockups as finished UI
- overhype features

---

## Phase 2 — Internal Alpha

Goal:
Test the first working reminder flow privately.

Required before leaving this phase:
- create reminder
- save reminder locally
- trigger notification
- snooze works
- done works
- app restart works

Devices:
- emulator
- Google Pixel 9 XL

---

## Phase 3 — Closed Testing

Goal:
Test QuietLog with a small group before public launch.

Focus:
- notification reliability
- shopping list usability
- game event flow
- crash-free daily use

Good tester types:
- Android users
- gamers
- people who disable most notifications
- privacy-focused users

---

## Phase 4 — Pre-Launch Store Presence

Goal:
Prepare the Google Play listing before full release.

Needed:
- app icon
- short description
- full description
- screenshots
- privacy policy
- feature graphic
- clear pricing direction

Store positioning:

> QuietLog is a private offline planner for personal reminders, shopping lists, and game events. No ads. No cloud. No account. No AI. No subscription.

---

## Phase 5 — Public Release

Only release publicly when reminder reliability feels boringly stable.

Minimum release quality:
- reminders trigger reliably
- no duplicate notifications
- snooze works
- done works
- reboot behavior tested
- app restart behavior tested
- no major crashes
- shopping lists stable
- game events stable enough for MVP

Main release principle:

> Trust is harder to regain than attention.

---

# Twitter / X Plan

## Posting Style

QuietLog posts should feel:
- calm
- honest
- transparent
- practical
- low-hype

Avoid:
- engagement bait
- fake urgency
- overpromising
- daily spam
- pretending unfinished features are done

Recommended cadence:

- 1–3 posts per week
- more posts only around real milestones

# Example First Posts

## Intro Post

Building QuietLog — a private offline planner for real life and gaming.

No ads.
No cloud.
No account.
No AI.
No subscription.

Just user-created reminders, shopping lists, and game events.

---

## Philosophy Post

Most apps use notifications to pull you back in.

QuietLog is built for the opposite:
only remind you about things you personally created.

Quiet by default.
Useful when needed.

---

## Reliability Post

For a reminder app, trust matters more than feature count.

QuietLog will not launch publicly until reminders are tested for:
- app restart
- snooze
- duplicate prevention
- reboot behavior
- real device reliability

---

## Shopping Feature Post

Small feature idea in QuietLog:

If you are shopping and stop checking items for a few minutes while items are still open, QuietLog can remind you:

“Still items left on your list.”

No GPS.
No store tracking.
Just simple local logic.

---

# Main Marketing Rule

Do not market QuietLog as exciting.

Market it as trustworthy.

QuietLog should feel like:

> A small app that respects your attention.
