# QuietLog — Commit Rules

## Goal

Commits should stay small, readable, and easy to understand later.

## Commit Style

Use simple conventional commits:

- chore:
- feat:
- fix:
- docs:
- refactor:
- test:
- style:

## Good Examples

- chore: initialize Android project
- feat: add reminder data model
- feat: add reminder notification prototype
- fix: prevent past reminders from scheduling
- docs: add notification architecture notes

## Bad Examples

- update stuff
- changes
- final fix
- big update

## Main Rule

One commit should explain one meaningful change.
